# Schema and Migrations

Alembic manages schema migrations. **Source of truth:** `src/backend/alembic/versions/`

| Revision | File | Description |
|----------|------|-------------|
| `001` | `001_initial_schema.py` | `users`, `tickets`, `comments` tables + PostgreSQL enums |

## Run migrations

```bash
cd src/backend
alembic upgrade head
```

## Rollback

```bash
cd src/backend
alembic downgrade base
```

## Generate new revision (after model changes)

```bash
cd src/backend
alembic revision --autogenerate -m "describe change"
```

Review autogenerated SQL before applying.
